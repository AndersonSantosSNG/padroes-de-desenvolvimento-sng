Este documento estabelece as diretrizes para a transferência de contexto técnico e de negócio entre engenheiros, garantindo a continuidade das operações e eliminando ilhas de conhecimento dentro das squads.

### 🧩 Rituais Obrigatórios

* **Onboarding Guiado:** Todo novo integrante deve passar por sessões de alinhamento com um padrinho (*buddy*) técnico para entender a arquitetura local e rodar o projeto em ambiente de desenvolvimento.
* **Handover Formal:** Caso um desenvolvedor mude de squad ou se desligue de um projeto, é obrigatória a criação de um documento de transição ou gravação de vídeo explicando o estado atual das entregas e pontos de atenção técnicos.

### 📌 Práticas Recomendadas

* **Pair Programming:** Incentivar a programação em par para disseminar o domínio de módulos complexos entre mais de um membro do time.
* **Tech Talks Internas:** Realizar alinhamentos periódicos rápidos (ex: quinzenais ou mensais) para apresentar novas ferramentas, refatorações significativas ou padrões adotados no projeto.
