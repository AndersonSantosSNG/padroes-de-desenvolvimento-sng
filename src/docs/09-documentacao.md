Este documento define as exigências mínimas de documentação técnica para garantir entendimento, rastreabilidade e sustentabilidade dos projetos.

### 📚 Documentação Obrigatória

* **README:** todo projeto deve possuir um README claro com objetivos, pré-requisitos, execução e estrutura.
* **Swagger/OpenAPI:** APIs devem ter documentação automatizada e acessível.
* **ADR:** decisões arquiteturais relevantes devem ser registradas em documentos formais.
* **Diagramas:** diagramas de arquitetura e fluxos principais devem ser mantidos atualizados.

### 📝 Diretrizes de Conteúdo

* A documentação deve ser objetiva, atualizada e alinhada com o estado real do projeto.
* Mudanças significativas devem refletir imediatamente nos documentos relevantes.
* Novos integrantes devem conseguir entender o projeto sem depender apenas do código.

### ⚠️ Padrão de Manutenção

* Documentação desatualizada é considerada um problema de qualidade.
* A responsabilidade de manter a documentação é compartilhada entre autores e mantenedores do projeto.
