Este documento define as boas práticas de escrita de código que devem ser adotadas em todos os projetos. O objetivo é promover legibilidade, previsibilidade e facilidade de manutenção.

### 🧼 Princípios Obrigatórios

* Aplicar **Clean Code** como base da escrita técnica.
* Escrever métodos curtos e com responsabilidade única.
* Utilizar nomes explícitos para classes, métodos, variáveis e funções.
* Evitar números mágicos e valores hardcoded sem contexto.
* Evitar comentários excessivos quando o código já estiver claro.

### 📏 Diretrizes de Qualidade

* O código deve ser simples, objetivo e fácil de compreender.
* Funções e classes devem ser pequenas e coesas.
* Repetição desnecessária deve ser eliminada por abstração adequada.
* O estilo de código deve ser consistente entre os membros da equipe.
* Mudanças devem preservar clareza sem introduzir complexidade desnecessária.

### 🚫 Práticas a Evitar

* Métodos longos com múltiplas responsabilidades.
* Nomes vagos como `data`, `obj` ou `valor` sem contexto.
* Trechos de código duplicados espalhados pelo projeto.
* Comentários que explicam o óbvio em vez de documentar intenção.
