Este documento define o ecossistema tecnológico padronizado para o desenvolvimento de software na companhia. A centralização da stack visa otimizar a sinergia entre as equipes, simplificar a manutenção dos ativos digitais e garantir a escalabilidade das soluções.

### 🚀 Tecnologias Core (Novos Projetos)
Para o desenvolvimento de novas soluções e serviços, as equipes devem adotar obrigatoriamente:

* **Backend Principal:** **C# (.NET)**, focado na construção de serviços robustos, APIs de alta performance e sistemas de missão crítica.
* **Frontend Principal:** **React + TypeScript**, garantindo interfaces dinâmicas, tipagem estática segura, componentização eficiente e alta manutenibilidade.

### 🏛️ Ecossistema de Suporte e Legados
Tecnologias homologadas para contextos específicos, automações e cenários de transição:

* **Sistemas Legados:** **PHP**, restrito à manutenção, sustentação e refatoração gradual de plataformas preexistentes.
* **Automações e Scraping:** **Python**, a linguagem oficial para o desenvolvimento de scripts de automação, rotinas de extração de dados (scraping) e inteligência de dados.

### 💾 Persistência de Dados
* **Bancos de Dados:** O padrão oficial e homologado para armazenamento relacional e não-relacional está em fase de definição pela governança de arquitetura (avaliando *SQL Server, MySQL e PostgreSQL*).

### OBS:
Para que seja utilizado uma stack que não esteja listada nesse documento será necessário ser feito um levantamento/estudo sobre a linguagem justificando a escolha.
