Este documento define o processo de revisão de código como parte essencial da garantia de qualidade e da redução de riscos técnicos.

### ✅ Checklist Mínimo

Antes de aprovar uma alteração, o reviewer deve validar:

* O código compila corretamente.
* Os testes relevantes executam com sucesso.
* A implementação segue a arquitetura definida.
* A nomenclatura e o estilo estão consistentes.
* Não há riscos claros de segurança.
* A mudança não introduz débito técnico desnecessário.

### 🔍 Boas Práticas no Review

* Comentar de forma objetiva e construtiva.
* Focar em qualidade, clareza e impacto da mudança.
* Verificar se a solução é proporcional ao problema.
* Solicitar melhorias quando a implementação comprometer manutenibilidade.

### 📌 Regra de Aprovação

* Pull requests devem ser revisados por pelo menos um colega qualificado.
* Alterações críticas devem passar por revisão adicional.
* Mudanças de alto impacto devem ser discutidas com a arquitetura ou governança técnica.
